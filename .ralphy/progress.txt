### Design decisions:
- Add columns directly to `phones` table (not separate metadata table) — all fields are 1:1
- Difficulty is computed from brand popularity + release recency, not stored
- Images stay in git (`config/public/phones/`) — too small for CDN overhead
- New columns are nullable for backwards-compatible migration
- `phone-data.json` remains single source of truth for seeding
